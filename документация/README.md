## Описание работы программы

Робот читает список документов из файлов (JSON или Excel), выбранных пользователем через графическую форму, и создаёт каждый документ в 1С через HTTP-сервис API (блок «Отправить запрос в 1С» Puzzle RPA). Логирует каждый шаг и корректно обрабатывает ошибки.

---

## Задача

**Кейс 2 — RPA-робот для создания документов в 1С по входным данным.**

Цель: по структурированным входным данным (JSON / Excel / CSV) автоматически создавать учётные документы (счета на оплату и др.) в 1С:Предприятие через HTTP-сервисы API, логировать результат и сохранять отчёт о работе.

---

## Платформа

Проект создан в **Puzzle RPA Studio 3.0.3**

## Архитектура проекта

Проект разбит на шесть компонентов-процессов. Каждый компонент отвечает за свою зону ответственности и предоставляет набор процедур, которые вызываются из других компонентов.

```
решение/Кейс2_СозданиеДокументов1С/
├── process/
│   ├── Main.json            # Точка входа, оркестрация
│   ├── Config.json          # Настройки подключения к 1С
│   ├── Logger.json          # Логирование событий
│   ├── FIleReader.json      # Чтение входных файлов
│   ├── FormHandler.json     # Пользовательская форма
│   └── 1C integration.json  # Создание документов в 1С
├── resource/
│   ├── forms/
│   │   └── mainForm.json    # Описание UI-формы
│   ├── orders.json          # Пример входных данных (JSON)
│   ├── orders.xlsx          # Пример входных данных (Excel)
│   └── logs.json            # Файл логов текущего запуска
├── logs/                    # Исторические логи по датам
└── meta/                    # Метаданные проекта Puzzle RPA
```

## Компоненты

### Config (`process/Config.json`)

Хранит и предоставляет настройки подключения к 1С. Работает с глобальными переменными.

| Процедура / Функция | Описание |
|---|---|
| `initConfig()` | Инициализирует переменные пустыми значениями (вызывается при старте) |
| `setupConfig(data)` | Заполняет конфиг из словаря с данными формы (`DBPathInput`, `loginInput`, `passwordInput`, `logCheckbox`) |
| `get1CConfig()` | Возвращает объект `{1C_DB_PATH, 1C_LOGIN, 1C_PASSWORD}` |
| `getLogConfig()` | Возвращает флаг `LOG_ENABLE` (bool) |

**Глобальные переменные:** `1C_DB_PATH`, `1C_LOGIN`, `1C_PASSWORD`, `LOG_ENABLE`

---

### Logger (`process/Logger.json`)

Ведёт лог работы робота в файл `resource/logs.json`. Логирование можно отключить флагом.

| Процедура / Функция | Описание |
|---|---|
| `getLogFile()` | Возвращает абсолютный путь к `logs.json` |
| `initLogger()` | Удаляет старый `logs.json` и создаёт новый с пустым массивом `logs` |
| `setupLogger(showLogs)` | Устанавливает флаг `SHOW_LOGS` (включить / выключить логирование) |
| `log(operation, status, message)` | Добавляет запись `{operation, status, message, time}` в `logs.json`; ничего не делает, если `SHOW_LOGS = false` |

**Глобальная переменная:** `SHOW_LOGS`

**Формат записи в лог:**
```json
{
  "operation": "Read file",
  "status": "success",
  "message": "file: orders.xlsx",
  "time": "23.05.2026 14:30:00"
}
```

---

### FIleReader (`process/FIleReader.json`)

Читает входные файлы и возвращает их содержимое в виде Python-объектов (словари / списки).

| Функция | Описание |
|---|---|
| `readJSON(path)` | Читает JSON-файл, логирует успех, возвращает объект |
| `readExcel(path, listName)` | Читает лист `listName` из Excel-файла, возвращает массив записей (JSON Records) |
| `readFile(path)` | Универсальная обёртка: определяет тип файла по расширению и вызывает нужную функцию. Поддерживает `.json`, `.xlsx`, `.csv`. При неизвестном расширении логирует ошибку |

Пути передаются относительными — компонент сам строит абсолютный путь через `get_executable_path`.

---

### FormHandler (`process/FormHandler.json`)

Показывает пользователю графическую форму и обрабатывает её результат.

| Процедура / Функция | Описание |
|---|---|
| `showForm()` | Открывает `forms/mainForm.json`, ждёт заполнения, возвращает список выбранных файлов (`selectedFiles`). При закрытии формы без подтверждения логирует `"form closed"` |
| `clickSetup(data)` | Обработчик кнопки «Настройки» в форме: вызывает `setupConfig(data)`, `setupLogger(getLogConfig())`, логирует параметры подключения (пароль маскируется `*`) |

Форма (`mainForm.json`) позволяет пользователю выбрать рабочие файлы, ввести путь к базе 1С, логин, пароль и переключить логирование.

---

### 1C integration (`process/1C integration.json`)

Отвечает за непосредственное взаимодействие с 1С через HTTP-сервис.

| Функция | Описание |
|---|---|
| `1CCreateDocument(data)` | Формирует запрос и отправляет его в 1С через блок `send_request_1c` (тип «Взаимодействие с данными», операция «Создать»). Принимает словарь с реквизитами и табличными частями документа |

Блок `send_request_1c` использует `1C_DB_PATH`, `1C_LOGIN`, `1C_PASSWORD` из Config.

**Структура данных, передаваемых в 1С:**
```json
{
  "Реквизиты": [
    {
      "Номер": "СЧ-001",
      "Дата": "2026-05-23",
      "Контрагент": "ООО Ромашка",
      "Сумма": 15000
    }
  ],
  "ТабличныеЧасти": [
    {
      "Наименование": "Товар А",
      "Количество": 2,
      "Цена": 5000,
      "Сумма": 10000
    }
  ]
}
```

---

### Main (`process/Main.json`)

Точка входа. Оркестрирует все остальные компоненты в нужном порядке.

**Последовательность выполнения:**

```
1. initConfig()          — инициализация настроек
2. initLogger()          — сброс лога
3. formResult = showForm()  — показ формы пользователю
4. Цикл по formResult:
   4.1. fileData = readFile(file)   — чтение файла
   4.2. [дальнейшая обработка — создание документов в 1С]
5. log("Programm ended", "success", "")
```

## Формат входных данных

Робот принимает файлы `.json`, `.xlsx` или `.csv`. Файлы выбираются пользователем через форму во время работы.

**Пример (`resource/orders.json`):**
```json
[
  {
    "document_type": "Документ.СчетНаОплату",
    "number": "СЧ-001",
    "date": "2025-04-01",
    "counterparty": "ООО Ромашка",
    "sum": 15000.00,
    "items": [
      {"product": "Товар А", "quantity": 2, "price": 5000, "sum": 10000},
      {"product": "Товар Б", "quantity": 1, "price": 5000, "sum": 5000}
    ]
  }
]
```

## Запуск

1. Открыть проект в **Puzzle RPA Studio 3.0.3** (`решение/Кейс2_СозданиеДокументов1С`).
2. Установить процесс `Main` как главный: *Проект → Главный процесс → Main*.
3. Нажать **F5** (или кнопку «Запустить»).
4. В открывшейся форме:
   - Указать путь к базе 1С, логин и пароль.
   - Выбрать файлы с данными для обработки.
   - Нажать «Запустить».
5. Логи выполнения сохраняются в `resource/logs.json` и в папке `logs/` (по датам).

## Описание работы программы

Робот читает входные данные из файла (`.json` или `.xlsx`), открывает браузер, авторизуется в веб-версии 1С:Бухгалтерия и автоматически создаёт документы через графический интерфейс системы. Заполняются поля контрагента, организации и табличная часть товаров. Для надёжности каждый шаг выполняется через XPath; если элемент не найден — автоматически применяется резервный поиск через машинное зрение (поиск по эталонному изображению). По завершении работы робот опционально проводит документы и сохраняет CSV-отчёт с результатом обработки каждого документа. Каждый шаг логируется в `logs.json`.

**MVP полностью реализует требования кейса:**
- чтение входных данных из `.json` и `.xlsx`;
- браузерная автоматизация с двойной защитой (XPath + машинное зрение);
- создание и заполнение документов в 1С;
- опциональное проведение каждого документа;
- формирование CSV-отчёта о результатах работы;
- детальное логирование в `logs.json`.

## Задача

**Кейс 2 — RPA-робот для создания документов в 1С по входным данным.**

Цель: по структурированным входным данным (JSON / Excel) автоматически создавать учётные документы (счета на оплату) в 1С:Предприятие через веб-интерфейс, используя браузерную автоматизацию Puzzle RPA, опционально проводить созданные документы, логировать результат и сохранять отчёт о работе в CSV.

## Компоненты

### Main (`process/Main.json` / `Main.py`)

Точка входа. Реализует полный цикл: чтение данных → авторизация → навигация → создание документа → заполнение → проведение → CSV-отчёт.

**Последовательность выполнения:**

```
1. Инициализировать Config и Logger
2. Показать форму выбора файла (FormHandler)
3. Прочитать входные данные (FIleHandler: JSON или Excel)
4. Открыть браузер Chrome
5. Перейти на URL базы 1С
6. Авторизация:
   6a. [XPath] Ввести логин → пароль → нажать «Войти»
   6b. [Fallback] Если XPath не найден → использовать машинное зрение (login_input.png, password_input.png, login_btn.png)
7. [Если] Появился диалог Google Password Manager → закрыть его (google_pass_close_btn.png)
8. [Если] 1С загрузилась (1с_ready_window.png, порог 0.8, таймаут 40 с):
   Для каждого документа из входных данных:
   8.1. Перейти Продажи → Счета покупателям
   8.2. Нажать «Создать»:
        [XPath] #form5_ФормаСоздать
        [Fallback] create_document_btn.png
   8.3. Заполнить «Контрагент»:
        [XPath] //*[contains(@id, "_Контрагент_i0")]
        [Fallback] counterparty_input.png
        → Enter (подтверждение из справочника)
   8.4. Заполнить «Организация»:
        [XPath] #form6_Организация_i0
        [Fallback] organization_field.png
        → Enter (подтверждение из справочника)
   8.5. Для каждой строки товара:
        – Нажать «Добавить»:
          [XPath] #form6_ТоварыДобавить
          [Fallback] add_item_btn.png
        – Ввести Номенклатуру → Enter
        – Ввести Количество:
          [XPath] #form6_ТоварыКоличество_i0
          [Fallback] quantity_item_input.png
          → Enter
        – Ввести Цену → Enter
   8.6. Если submit = true → «Провести и закрыть» (submit_and_save_btn.png)
        Иначе → Ctrl+S (сохранить без проведения / save_btn.png)
   8.7. Записать результат в report.csv (document, status)
9. Показать уведомление «end»
```

### Config (`process/Config.json` / `Config.py`)

Хранит и предоставляет настройки подключения к 1С.

| Функция | Описание |
|---|---|
| `initConfig()` | Инициализирует все переменные пустыми значениями |
| `setupConfig(data)` | Заполняет конфиг из словаря формы (`DBPathInput`, `loginInput`, `passwordInput`, `logCheckbox`) |
| `get1CConfig()` | Возвращает `{1C_DB_PATH, 1C_LOGIN, 1C_PASSWORD}` |
| `getLogConfig()` | Возвращает флаг `LOG_ENABLE` |

**Глобальные переменные:** `my_1C_DB_PATH`, `my_1C_LOGIN`, `my_1C_PASSWORD`, `LOG_ENABLE`

---

### Logger (`process/Logger.json` / `Logger.py`)

Ведёт лог работы в файл `resource/logs.json`.

| Функция | Описание |
|---|---|
| `getLogFile()` | Возвращает абсолютный путь к `logs.json` |
| `initLogger()` | Удаляет старый `logs.json`, создаёт новый с пустым массивом |
| `setupLogger(showLogs)` | Устанавливает флаг `SHOW_LOGS` |
| `log(operation, status, message)` | Добавляет запись в лог; пропускает, если `SHOW_LOGS = false` |

**Формат записи:**
```json
{
  "operation": "Read JSON",
  "status": "success",
  "message": "file: orders.json",
  "time": "01.04.2025 14:30:00"
}
```

---

### FIleHandler (`process/FIleHandler.json` / `FIleHandler.py`)

Читает входные файлы и возвращает содержимое как Python-объекты.

| Функция | Описание |
|---|---|
| `readFile(path)` | Универсальная обёртка: по расширению файла вызывает нужный метод |
| `readJSON(path)` | Читает `.json`, логирует, возвращает объект |
| `readExcel(path, listName)` | Читает первый лист из `.xlsx`, возвращает массив записей (JSON Records); логирует |

При чтении Excel автоматически определяется первый лист (`get_sheet_names_excel`). Плоская структура строк (одна строка = один товар) группируется по полю `number` для восстановления вложенного формата документов.

---

### FormHandler (`process/FormHandler.json` / `FormHandler.py`)

Показывает пользователю графическую форму выбора файла и настроек подключения.

| Функция | Описание |
|---|---|
| `showForm()` | Открывает `forms/mainForm.json`, возвращает список выбранных файлов (`selectedFiles`). При закрытии без подтверждения — логирует ошибку |
| `clickSetup(data)` | Обработчик кнопки «Настройки»: вызывает `setupConfig` + `setupLogger`, логирует параметры (пароль маскируется `***`) |

---

### 1C integration (`process/1C integration.json` / `robo_229.py`)

Вспомогательный компонент с HTTP API-подходом к созданию документов.

| Функция | Описание |
|---|---|
| `my_1CCreateDocument(data)` | Отправляет запрос в 1С через `send_request_1c` (тип 3 — «Взаимодействие с данными», режим `write`) |

Функция принимает словарь:
```python
{
    "object": "Документ.СчетНаОплату",
    "data": {
        "Реквизиты": [{"Номер": ..., "Дата": ..., "Контрагент": ..., "Сумма": ...}],
        "ТабличныеЧасти": [{"Наименование": ..., "Количество": ..., "Цена": ..., "Сумма": ...}]
    }
}
```

## Машинное зрение (резервный метод)

Для каждого ключевого действия предусмотрено два уровня поиска элементов:

1. **Основной** — поиск по XPath/CSS-селектору через CDP (быстро, точно).
2. **Резервный** — поиск по эталонному изображению через `expect_screen_images` / `click_on_picture` (устойчив к изменениям идентификаторов).

## Формат входных данных

Входной файл выбирается пользователем через форму. Поддерживаются `.json` и `.xlsx`.

### `resource/orders.json`
```json
[
  {
    "document_type": "Документ.СчетНаОплату",
    "section":       "Продажи",
    "subsection":    "Счета покупателям",
    "number":        "СЧ-001",
    "date":          "2025-04-01",
    "counterparty":  "Кафе \"Аполлон\"",
    "organization":  "Магазин №23",
    "sum":           13500.00,
    "submit":        false,
    "items": [
      {"product": "Ассорти", "quantity": 5, "price": 1500, "sum": 7500},
      {"product": "Белочка", "quantity": 3, "price": 2000, "sum": 6000}
    ]
  }
]
```

| Поле | Тип | Описание |
|---|---|---|
| `document_type` | string | Тип документа 1С (`Документ.СчетНаОплату`) |
| `section` | string | Раздел меню 1С (напр. `Продажи`) |
| `subsection` | string | Подраздел меню 1С (напр. `Счета покупателям`) |
| `number` | string | Номер счёта |
| `date` | string | Дата документа (`YYYY-MM-DD`) |
| `counterparty` | string | Контрагент (должен существовать в справочнике 1С) |
| `organization` | string | Организация (должна существовать в справочнике 1С) |
| `sum` | number | Итоговая сумма документа |
| `submit` | bool | `true` — провести документ, `false` — только сохранить |
| `items[]` | array | Табличная часть: товары с количеством, ценой и суммой |

### `resource/orders.xlsx`

Плоская таблица — одна строка на товар, поля документа повторяются в каждой строке:

| Колонки документа (1–9) | Колонки товара (10–13) |
|---|---|
| document_type, section, subsection, number, date, counterparty, organization, sum, submit | product, quantity, price, item_sum |

Для восстановления вложенной структуры строки группируются по полю `number`.

## Формат CSV-отчёта

После завершения обработки всех документов результаты сохраняются в `resource/report.csv`.

| Поле | Описание |
|---|---|
| `document` | Идентификатор документа (номер счёта) |
| `status` | Результат: `success` или `error` |

Пример:
```
document,status
СЧ-001,success
СЧ-002,error
```

## Запуск

1. Открыть проект в **Puzzle RPA Studio 3.0.3** (`решение/Кейс2_СозданиеДокументов1С`).
2. Установить процесс `Main` как главный: *Проект → Главный процесс → Main*.
3. Нажать **F5** (или кнопку «Запустить»).
4. В открывшейся форме:
   - Выбрать входной файл (`orders.json` или `orders.xlsx`).
   - При необходимости изменить настройки подключения (кнопка «Настройки»).
5. Нажать «Подтвердить» — робот автоматически:
   - Откроет браузер Chrome.
   - Авторизуется в 1С (XPath; при сбое — машинное зрение).
   - Создаст и заполнит документы по данным из файла.
   - Проведёт документы с `"submit": true`.
   - Сохранит `resource/report.csv` с результатами.
6. Логи выполнения сохраняются в `resource/logs.json`.

## Описание работы программы

Робот читает входные данные из файла (JSON или Excel), открывает браузер, авторизуется в веб-версии 1С:Бухгалтерия и автоматически создаёт документ «Счёт покупателю» через графический интерфейс системы. Заполняются поля контрагента, организации и табличная часть товаров. Каждый шаг логируется.

---

## Задача

**Кейс 2 — RPA-робот для создания документов в 1С по входным данным.**

Цель: по структурированным входным данным (JSON / Excel) автоматически создавать учётные документы (счета на оплату) в 1С:Предприятие через веб-интерфейс, используя браузерную автоматизацию Puzzle RPA, логировать результат и сохранять отчёт о работе.

## Компоненты

### Main (`process/Main.json` / `Main.py`)

Точка входа. Реализует полный цикл создания документа в 1С через браузерную автоматизацию.

**Последовательность выполнения:**

```
1. Открыть браузер Chrome
2. Перейти на https://demo1c.mkskom.ru/puzzle_buh_corp_8.3
3. Ввести логин и пароль → нажать «Войти»
4. [Если] Появился диалог Google Password Manager → закрыть его
5. [Если] 1С загрузилась (определяется по эталонному изображению):
   5.1. Перейти Продажи → Счета покупателям
   5.2. Нажать «Создать»
   5.3. Заполнить поле «Контрагент»  (Enter для подтверждения из справочника)
   5.4. Заполнить поле «Организация» (Enter для подтверждения из справочника)
   5.5. Для каждой строки товара:
        – Нажать «Добавить» в табличной части
        – Ввести номенклатуру (Enter)
        – Ввести количество (Enter)
        – Ввести цену (Enter)
        – Нажать Insert для перехода к следующей строке
   5.6. Сохранить документ (Ctrl+S)
6. Показать уведомление «end»
```

### Config (`process/Config.json` / `Config.py`)

Хранит и предоставляет настройки подключения к 1С.

| Функция | Описание |
|---|---|
| `initConfig()` | Инициализирует все переменные пустыми значениями |
| `setupConfig(data)` | Заполняет конфиг из словаря формы (`DBPathInput`, `loginInput`, `passwordInput`, `logCheckbox`) |
| `get1CConfig()` | Возвращает `{1C_DB_PATH, 1C_LOGIN, 1C_PASSWORD}` |
| `getLogConfig()` | Возвращает флаг `LOG_ENABLE` |

**Глобальные переменные:** `my_1C_DB_PATH`, `my_1C_LOGIN`, `my_1C_PASSWORD`, `LOG_ENABLE`

---

### Logger (`process/Logger.json` / `Logger.py`)

Ведёт лог работы в файл `resource/logs.json`.

| Функция | Описание |
|---|---|
| `getLogFile()` | Возвращает абсолютный путь к `logs.json` |
| `initLogger()` | Удаляет старый `logs.json`, создаёт новый с пустым массивом |
| `setupLogger(showLogs)` | Устанавливает флаг `SHOW_LOGS` |
| `log(operation, status, message)` | Добавляет запись в лог; пропускает, если `SHOW_LOGS = false` |

**Формат записи:**
```json
{
  "operation": "Read JSON",
  "status": "success",
  "message": "file: orders.json",
  "time": "01.04.2025 14:30:00"
}
```

---

### FIleReader (`process/FIleReader.json` / `FIleReader.py`)

Читает входные файлы и возвращает содержимое как Python-объекты.

| Функция | Описание |
|---|---|
| `readFile(path)` | Универсальная обёртка: по расширению вызывает нужный метод |
| `readJSON(path)` | Читает `.json`, логирует, возвращает объект |
| `readExcel(path, listName)` | Читает лист из `.xlsx`/`.csv`, возвращает массив записей (JSON Records). Автоматически берёт первый лист |

---

### FormHandler (`process/FormHandler.json` / `FormHandler.py`)

Показывает пользователю графическую форму.

| Функция | Описание |
|---|---|
| `showForm()` | Открывает `forms/mainForm.json`, возвращает список выбранных файлов (`selectedFiles`). При закрытии без подтверждения — логирует ошибку |
| `clickSetup(data)` | Обработчик кнопки «Настройки»: вызывает `setupConfig` + `setupLogger`, логирует параметры (пароль маскируется `***`) |

---

### 1C integration (`process/1C integration.json` / `robo_229.py`)

Вспомогательный компонент с HTTP API-подходом к созданию документов.

| Функция | Описание |
|---|---|
| `1CCreateDocument(data)` | Отправляет запрос в 1С через `send_request_1c` (тип 3 — «Взаимодействие с данными», режим `write`) |

Функция принимает словарь:
```python
{
    "object": "Документ.СчетНаОплату",
    "data": {
        "Реквизиты": [{"Номер": ..., "Дата": ..., "Контрагент": ..., "Сумма": ...}],
        "ТабличныеЧасти": [{"Наименование": ..., "Количество": ..., "Цена": ..., "Сумма": ...}]
    }
}
```

## Формат входных данных

Входной файл выбирается пользователем через форму. Поддерживаются `.json` и `.xlsx`.

**`resource/orders.json`:**
```json
[
  {
    "document_type": "Документ.СчетНаОплату",
    "section": "Продажи",
    "subsection": "Счета покупателям",
    "number": "СЧ-001",
    "date": "2025-04-01",
    "counterparty": "Кафе \"Аполлон\"",
    "organization": "Магазин №23",
    "sum": 13500.00,
    "items": [
      {"product": "Ассорти",  "quantity": 5, "price": 1500, "sum": 7500},
      {"product": "Белочка",  "quantity": 3, "price": 2000, "sum": 6000}
    ]
  }
]
```

| Поле | Описание |
|---|---|
| `document_type` | Тип документа 1С |
| `section` | Раздел меню 1С (напр. `Продажи`) |
| `subsection` | Подраздел меню 1С (напр. `Счета покупателям`) |
| `number` | Номер счёта |
| `date` | Дата документа (`YYYY-MM-DD`) |
| `counterparty` | Контрагент (должен существовать в справочнике 1С) |
| `organization` | Организация (должна существовать в справочнике 1С) |
| `sum` | Итоговая сумма документа |
| `items[]` | Табличная часть: товары с количеством, ценой и суммой |

## Запуск

1. Открыть проект в **Puzzle RPA Studio 3.0.3** (`решение/Кейс2_СозданиеДокументов1С`).
2. Установить процесс `Main` как главный: *Проект → Главный процесс → Main*.
3. Нажать **F5** (или кнопку «Запустить»).
4. Робот автоматически:
   - Откроет браузер Chrome
   - Авторизуется в 1С
   - Создаст и заполнит документ по данным из файла
5. Логи выполнения сохраняются в `resource/logs.json`.
